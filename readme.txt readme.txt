=== หลักการทำงานของระบบ Batman_gRPC-REST ===

โปรเจกต์นี้ประกอบด้วย 3 บริการหลัก (Microservices) ที่ทำงานร่วมกันผ่านโปรโตคอล REST และ gRPC:

1. Service A (The Hub / Server):
   - ทำหน้าที่เป็นศูนย์กลางของระบบ
   - เปิดบริการ REST API ที่พอร์ต 8000 (สำหรับ HTTP requests)
   - เปิดบริการ gRPC Server ที่พอร์ต 50051 (สำหรับเชื่อมต่อระหว่าง Service)
   - เก็บข้อมูลหลักและประมวลผลคำขอจาก Service อื่นๆ

2. Service B (gRPC Client):
   - ทำหน้าที่เป็น Client ที่สื่อสารผ่าน gRPC
   - ส่งคำขอไปยัง Service A (พอร์ต 50051) เพื่อดึงข้อมูลหรือสั่งการ
   - ข้อดี: การรับส่งข้อมูลรวดเร็วมากและมีโครงสร้างที่ชัดเจน (Strongly Typed)

3. Service C (REST Client):
   - ทำหน้าที่เป็น Client ที่สื่อสารผ่าน REST API (HTTP)
   - ส่งคำขอไปยัง Service A (พอร์ต 8000)
   - ข้อดี: ใช้งานง่ายและรองรับการเชื่อมต่อกับเว็บเบราว์เซอร์หรือเครื่องมือทั่วไปได้ดี

สรุปการไหลของข้อมูล:
- เมื่อต้องการข้อมูลผ่าน gRPC -> Service B เรียกหา Service A
- เมื่อต้องการข้อมูลผ่าน REST -> Service C เรียกหา Service A
- ทั้งหมดทำงานอยู่ภายใต้ Docker Compose เพื่อให้ง่ายต่อการรันและจัดการเครือข่าย
